📦 Project: Minimal Go + Hetzner Infra Bootstrap
🎯 Goal

A minimal but professional infrastructure setup that can:

Provision a Hetzner VPS automatically
Deploy a Go “Hello World” service
Run everything from a single command (make bootstrap)
Keep secrets and config clean and separated
Be fully reproducible from a Git repository
🧱 Architecture
1. Infrastructure (Terraform)
   Creates a Hetzner Cloud VPS (Ubuntu 22.04)
   Injects SSH key automatically
   Outputs server IP
   Uses cloud-init for base setup (UFW + directories)
2. Server bootstrap (cloud-init)
   Installs minimal packages (UFW)
   Opens required ports (SSH + 8080)
   Prepares /opt/app directory
   Optionally writes systemd environment file
3. Application (Go)
   Simple HTTP server
   Listens on port from environment (APP_PORT)
   Returns “hello world”
   ⚙️ Deployment model
   Makefile is the only entrypoint

Core commands:

make bootstrap
runs Terraform
builds Go binary
deploys to server
restarts systemd service
prints public URL
make deploy
rebuilds + redeploys app only
make down
destroys infrastructure
🔐 Secrets & configuration
Local development:
.env (NOT committed)
.env.example (committed)
Production:
/etc/helloworld.env on server
loaded via systemd EnvironmentFile
No secrets stored in Git or Terraform state
🧩 Design principles
Minimal manual steps (ideally only make bootstrap)
Reproducible infrastructure from Git
Clear separation:
Infra (Terraform)
Deploy logic (Makefile + script)
Runtime config (systemd env file)
No Kubernetes, no Vault, no heavy tooling (intentionally)
🚀 Current maturity level
Single-server production-grade baseline
Not yet HA / scalable cluster
Focus: learning + clean DevOps foundation
